\chapter{B-rep 几何序列化方法}
\chaptermark{B-rep 几何序列化方法}

本章介绍把 B-rep 模型转换为可训练的 token 序列的整套预处理流程，主要内容包括：数据预处理总体流程、B-rep 几何 / 拓扑分解、面 / 边 NURBS 采样、坐标量化、统一词表、面边分块序列结构，以及面 / 边的拓扑感知排序策略。

\section{数据预处理总体流程}

整套预处理从 STEP 文件出发，经五个阶段抵达可训练的序列文件：STEP 解析 $\to$ 拓扑分解 $\to$ 面 / 边几何采样 $\to$ 坐标量化 $\to$ 拓扑感知排序与序列拼装。其中 STEP 解析与拓扑分解通过 OpenCASCADE 完成。

本文实现中保留了原始 NURBS 采样版、量化版以及服务器侧镜像脚本，三者在核心采样、量化与排序规则上保持一致。最终训练数据包含 64\,351 个训练样本与 6\,096 个验证样本，数据侧统一词表规模为 4\,053，模型训练时额外加入 PAD，因此模型 \texttt{vocab\_size} 为 4\,054。正文聚焦序列化规则、数据统计与实验结果，不逐一展开脚本、数据文件和日志路径。

预处理阶段在工程上要同时满足三类约束。其一，输出序列必须具有足够规则的语法骨架，让 decoder-only Transformer 能在大量样本中反复看到相同的位置类型模式，从而学到稳定的下一 token 分布，这要求 START / SEP / END 以及固定长度的面段、边段在所有样本中长度一致、出现位置一致。其二，序列中需要保留可重建所需的几何与拓扑信息，而不只是用于可视化的外观采样点。单纯把面表面采样几个点是不够的，还要附带能够把它和其他面“接起来”的面索引信息。其三，单个样本长度要控制在可训练范围内，使 4\,096 token 的上限能覆盖绝大多数 DeepCAD 子集样本，否则要么牺牲训练效率，要么大面积过滤导致样本数不足。结合这三类约束，本文没有直接序列化完整 STEP 实体中的所有底层字段，而是选取了“面采样点 + 面索引”和“边邻接面索引 + 边采样点”作为主要信息载体：前者负责告诉模型“这个面长什么样、是该样本中的哪个面”，后者负责告诉模型“这条边连接哪两个面、它的几何走向是怎样的”。

\subsection{数据清洗与过滤原则}

预处理阶段并非简单地把所有 STEP 文件无差别转为序列。公开 CAD 数据集中常见的异常情况包括：STEP 文件本身已损坏导致内核无法读取，实体存在拓扑退化（如零面或零边），面数或边数远超本文模型设定的局部面索引上限，以及几何内核读入后得到的对象不属于本文可处理的实体类型。这些样本若直接进入训练集，会让模型把大量注意力分配给“异常格式如何被部分解析”这类与几何无关的模式，进而显著增加 §5.4 中观察到的 S1 失败概率。本文在数据转换时按三步过滤：首先要求 OpenCASCADE 能够成功读入文件并得到可遍历的几何拓扑对象，其次要求该实体能够提取出非空的 Face 集合和 Edge 集合，且面数不超过本文局部面索引上限 50，最后要求依规则拼出的 token 序列长度不超过 4\,096。不满足任一条件的样本直接舍弃，不会进入最终训练文件。

这套过滤策略的取向是保证训练样本的结构一致性，而不是追求数据量最大。对于本文这样的自回归模型，一个 batch 内的样本需要经过 PAD 对齐，过长序列不仅占用显存，还会降低 batch 中有效 token 的比例，影响有效吞吐，而拓扑明显异常的样本会让模型学习到不稳定的边界模式，最终增加生成阶段的 S1 失败概率。出于这两点考虑，本文宁愿舍弃少量复杂或异常样本，也要保证保留下来的序列在 START、SEP、END、面段与边段上具有稳定规律。

这种过滤口径也决定了本文结论的适用范围。被排除的样本通常对应复杂度更高或导出质量较差的 CAD 文件，它们并不能代表方法完全无法处理，只是超出了当前原型系统设定的训练边界。后续若希望提升覆盖面，主要有两条路线：一是直接提高上下文长度并配合 sliding-window 或线性注意力等机制，以容纳更多面和边，二是在更深层引入结构压缩，例如把局部面片预编码为更短的几何 token，再在高层序列中生成拓扑关系。本文先采用严格过滤而非这两类扩展，是为了让本科毕设阶段的实验建立在可解释、可复现的数据基础上，避免方法变量与数据变量同时变化导致结果难以归因。

\section{B-rep 几何拓扑分解方法}

\subsection{拓扑元素提取}

读入一个 STEP 文件后，使用 OpenCASCADE 的拓扑遍历接口按 Face、Edge、Vertex 顺序遍历拓扑层级，得到面集合 $\{f_k\}_{k=1}^{F}$ 与边集合 $\{e_j\}_{j=1}^{E}$，并维护面—边邻接表：对每个面记录其所有 Wire 上的边索引，对每个边记录其相邻的两个面索引。这一阶段不修改任何几何，只构建索引表与邻接图。

邻接表是后续排序与重建的核心中间结构。对每条边而言，若其恰好被两个面共享，则该边在拓扑上承担面间连接作用，若只关联一个面，则可能是开放边或边界异常，需要在重建阶段谨慎处理。DeepCAD 子集中的多数零件属于封闭机械实体，边通常对应两个邻接面，但预处理代码仍需兼容边界异常、退化边和读取失败等情况。本文在序列化时用两个面索引 token 表示边的邻接关系，使模型能够在生成边段时同时输出“这条边连接哪些面”和“这条边的几何位置”。

该阶段还会为每个样本建立局部面索引，而不是使用数据集全局面编号。局部索引的取值范围被限制在 $[0,49]$，对应最多 50 个面，这样可以把面索引与坐标 token 放入同一个固定词表中。对于面数过多或序列长度过长的样本，数据加载阶段通过长度过滤将其排除，保证训练 batch 内序列张量形状可控。该取舍会损失部分复杂零件，但有利于本科毕设阶段构建稳定可复现实验。

\subsection{NURBS 曲面参数域采样}

对每个面 $f_k$，在其参数域 $[u_{\min}, u_{\max}] \times [v_{\min}, v_{\max}]$ 上做均匀网格采样，得到 16 个三维点，作为该面在序列中的几何样本：

\begin{equation}
    \mathcal{P}_k = \{ S_k(u_a, v_b) \mid a, b = 0, 1, 2, 3 \}.
    \label{eq:face-uv-sampling}
\end{equation}

为减小参数化任意性带来的分布偏差，预处理阶段对每个面 UV 方向做 canonical sorting：按面包围盒主轴对 $u$、$v$ 方向重新排序。该 16 个三维点共贡献 48 个标量坐标 token，与 1 个面索引 token 一起构成长度为 49 的面段（详见 §3.4）。

$4 \times 4$ 的 UV 采样网格是在表达能力与序列长度之间的工程折中。采样点过少会让曲面拟合阶段难以恢复出具有足够约束的 NURBS 面：3 阶 B 样条曲面至少需要每方向 4 个控制点，再粗就只能退化为平面，而采样点过多会让每个 Face 的 token 数线性增长，按 16 提升到 25 时序列长度直接增加 56\%，长序列注意力开销与训练显存也随之上升。在 DeepCAD 子集中，多数机械零件的面属于平面、圆柱面、圆锥面或简单过渡曲面，16 个采样点已能覆盖其主曲率变化与曲面边界，对于高曲率自由曲面或带小特征的面片，该密度的不足会在重建阶段以拟合误差的形式体现出来，最终落入 §5.4 的 S2 类无效样本。本文在第~5 章把这一密度选择视为已知的容差源之一，没有把它作为独立超参数做扫描——这是因为同时调整采样密度、序列上限和模型容量将使变量过多，超出本科毕设的可控范围。

UV 参数域自身并不总是稳定。相同的几何曲面可能因 CAD 内核导出方式不同而拥有不同的参数起点、方向或裁剪边界。如果不做规范化，同一形状在序列中可能对应多个等价排列，从而增加模型学习负担。本文采用的 canonical sorting 并不能完全消除参数化任意性，但可以在工程上减少明显的方向翻转和坐标展开差异，使同类面在训练集中呈现更一致的 token 模式。

\subsection{NURBS 曲线弧长采样}

对每条边 $e_j$，按弧长在 $[0, L_j]$ 上均匀取参数点并求值，得到 4 个三维点作为该边在序列中的几何样本。弧长采样比参数域均匀采样更稳健，对参数化变形不敏感，对短边和长边都能给出可比较的采样密度。Edge 同样做 canonical sorting：按起止顶点的坐标顺序统一方向。该 4 个三维点共贡献 12 个标量坐标 token，与 2 个面索引 token（两端邻面）一起构成长度为 14 的边段（详见 §3.4）。

边采样在 B-rep 重建中承担“缝合约束”的作用。面采样点能够拟合出曲面片，但曲面片之间如何相交、交线在哪里、哪些面应当共享边界，需要通过边段提供的邻接关系和曲线采样点共同决定。本文没有直接生成顶点表，而是通过每条边的 4 个采样点隐式表达端点和中间形状。对直线边而言，4 个点存在冗余，对圆弧或一般样条边而言，中间点可以提供曲率线索，有助于拟合曲线。


\section{坐标量化与词表构建}

\subsection{标量量化方法}

所有采样点已经过单位化（缩放到 $[-1, 1]^3$）。对每个标量分量 $x \in [-1, 1]$ 应用均匀量化：

\begin{equation}
    q(x) = \left\lfloor \frac{x + 1}{2} \cdot N_q \right\rfloor + \text{offset}, \quad N_q = 4000,\ \text{offset} = 50.
    \label{eq:quantization}
\end{equation}

量化粒度约 $5 \times 10^{-4}$，对 1\,m 尺度的零件相当于 0.5\,mm 精度，对小特征也能保留毫米级细节。


本文将坐标统一缩放到 $[-1,1]^3$ 后再量化，目的是消除不同零件整体尺寸差异对训练的影响。若直接在原始物理尺度下量化，大尺寸零件会占据更大的坐标范围，小尺寸零件的细节则可能被压缩到很少的 token 区间内。归一化后，模型主要学习形状结构和相对几何关系，真实工程尺寸可在后处理阶段通过尺度恢复或条件输入进一步扩展，本文原型暂不处理尺寸条件生成。

量化误差可以用步长直接估计。由于坐标范围被压缩到 $[-1,1]$，4000 级均匀量化对应步长

\begin{equation}
    \Delta = \frac{2}{N_q}, \quad |\epsilon_q| \le \frac{\Delta}{2}.
    \label{eq:quantization-error}
\end{equation}

当 $N_q = 4000$ 时，单个标量分量的最大舍入误差约为 $2.5 \times 10^{-4}$。若把归一化空间还原到某一实际尺度，该误差会随零件包围盒尺寸线性放大：例如对包围盒边长 100 mm 的零件，单分量量化误差约为 0.025 mm，对 1 m 尺度的零件约为 0.25 mm。这一误差界说明本文方法的几何精度主要受两个因素影响：其一是采样点是否足以表达原始曲面的主要形态，其二是反量化后的采样点能否被后续 B 样条曲面拟合过程稳定吸收。量化本身并不是唯一误差来源，但它的上界是闭式可计算的，便于与 VQ-VAE 码本压缩中“码本索引—潜变量—解码器”三层级联误差区分开来。后者难以单独度量，前者只需查表即可。换言之，本文用更长的序列换来了几何误差源的可解释性。

均匀量化的局限在于它并不考虑不同坐标区域的几何重要性。尖角、短边和薄壁结构对局部误差更敏感，量化误差稍大就可能让相邻面之间的边界出现可见错位，而大平面区域即使误差稍大也未必影响 BRepCheck 合法性。本文选择均匀量化主要出于两点考虑：一是实现简单、反量化确定且不需要引入额外的可学习模块，便于在原型系统中追溯误差源，二是数据集中复杂局部特征所占比例较低，先用均匀量化跑通整条链路再做改进，比一开始就引入复杂量化策略更稳妥。若未来面向更高精度 CAD 生成，可以考虑非均匀量化、自适应局部尺度，或者把曲率信息纳入采样密度分配，使 token 预算更多地分配在几何变化剧烈的位置。

\subsection{统一词表设计}

为避免混淆，本文后续固定使用两个口径：数据词表（或序列词表）指预处理脚本写入原始 1D token 序列的有效 token 集，大小为 4\,053，模型词表指训练时 Embedding 表和输出层覆盖的 token 集，大小为 4\,054，等于数据词表再加 1 个 PAD。

数据侧的统一词表共 4\,053 个 token，按三段式布局：

\begin{itemize}
    \item 面索引段：$[0, 49]$，共 50 个 token，用于编码每个面在当前样本中的序号，
    \item 量化坐标段：$[50, 4049]$，共 4\,000 个 token，对应\autoref{eq:quantization} 中 $q(x)$ 的取值范围，
    \item 结构 token 段：$\text{START}=4050$，$\text{SEP}=4051$，$\text{END}=4052$。
\end{itemize}

数据词表大小为 $50 + 4000 + 3 = 4053$。模型侧额外引入一个 PAD token（id $= 4053$）用于批内序列长度对齐，因此模型输入输出词表规模为 4\,054，即数据词表大小加上 PAD。训练日志中同步记录数据词表、模型词表与 PAD 编号，用于复现实验配置。若下文单独讨论“统一词表”且未特别注明，均指不含 PAD 的数据侧统一词表。

这种三段布局让自回归模型可以从 logit 分布上直接识别当前位置应该输出哪一段，避免“几何 token 与控制 token 混淆”。


\section{面边分块序列结构}

\subsection{整体序列结构}

一个完整样本的 token 序列以 START 起、END 止，中间由 SEP 切分为“面段”与“边段”两部分：

\begin{verbatim}
START  [face_1 block] [face_2 block] ... [face_F block]
SEP    [edge_1 block] [edge_2 block] ... [edge_E block]  END
\end{verbatim}

    \centering
    \includegraphics[width=1\textwidth]{插图/fig3-3_seq_format.pdf}
    \caption{点级 token 序列结构示意图。完整序列由 START 开始，先顺序排列若干面段，再由 SEP 切换到边段，最后以 END 终止。每个面段含 16 个三维采样点对应的 48 个坐标 token 和 1 个面索引 token，每个边段含 2 个邻接面索引 token 和 4 个三维采样点对应的 12 个坐标 token。}
    \label{fig:sequence-format}
\end{figure}

\subsection{面分块的几何与索引编码}

每个面段长度记为 $B_f = 49$ 个 token：前 48 个 token 是当前面 16 个三维采样点按 $(x, y, z)$ 展平后的标量坐标 token，最后 1 个 token 是面索引 token。分组实现将 48 个坐标 token 按 3 个一组组织为 16 个 $[x, y, z]$ 三元组，再追加一个 $[\text{idx}, \text{PAD}, \text{PAD}]$ 的面索引三元组。

\subsection{边分块的邻接与几何编码}

每个边段长度记为 $B_e = 14$ 个 token：前 2 个 token 是该边两端邻面的面索引 token，后 12 个 token 是当前边 4 个三维采样点按 $(x, y, z)$ 展平后的标量坐标 token。分组时，两个面索引分别被组织为 $[\text{idx}, \text{PAD}, \text{PAD}]$ 三元组，12 个坐标 token 则被组织为 4 个 $[x, y, z]$ 三元组并追加在后。

\subsection{三元组化分组表达}

三元组化分组步骤把原始 1D token 序列重新组织为形状 $(L, 3)$ 的张量：每一“行”是一个三元组，几何点位置上的行对应 $[x, y, z]$（三个坐标 token），面索引或特殊符号位置上的行则用 PAD 填充为 $[\text{token}, \text{PAD}, \text{PAD}]$。这种统一形状的好处是：训练侧无论窗口大小为多少，所有样本都共用同一种张量布局，可以直接进入主干 embedding，推理侧每一步的生成单位也是一个三元组，方便与面段、边段的边界对齐。本文主实验的窗口大小为 1，每一行作为标准 next-token 的单步预测目标，模型每一步只预测一个新 token，PAD 位置由模型直接写入而不进入 softmax。代码中也保留了 $W > 1$ 的窗口化分支以支持点级三分量联合建模（详 §4.4），但与本节描述的标准布局不冲突。两者共享同一份 $(L, 3)$ 张量入口，只是后续如何把窗口内的多个 token 共同投入主干的方式不同。

在本文主实验中，$W = 1$ 的选择更加稳健。它把训练目标退化为最标准的 next-token 预测，模型结构、损失函数和推理逻辑都较清晰，同时避免在窗口内部再引入一层局部 decoder 后难以区分不同模块贡献。由于第~5 章结果已经能达到任务书有效率目标，本文将窗口化联合建模作为后续工作保留，而不把未启用分支写入实测结论。

\section{序列边界与控制符号}

\subsection{边界控制符号}

START 出现在序列首部，标记自回归起点，SEP 分隔面段与边段，END 标志生成终止。这三个 token 在数据词表中已经显式编码。

\subsection{填充符号与训练约束}

PAD（id $= 4053$）是模型侧引入的填充 token，用于两类用途：(a) 批内不同序列长度的对齐填充，对应位置在注意力掩码中标记为无效，(b) 在面索引或特殊符号位置上的 $t_1$、$t_2$ 分量填充（仅在窗口化分支启用时使用）。训练阶段对 PAD 位置的 label 设为 $-100$，并在交叉熵损失中跳过这些位置。推理阶段当生成位置类别决定应该输出 PAD 时直接写入 PAD，不进入 softmax 采样。这一约束让模型只在“需要预测有效 token”的位置参与梯度。

\subsection{序列格式一致性校验}

在写入训练文件前，预处理脚本会对每条样本执行基本格式检查：序列必须以 START 开始，以 END 结束，START 与 END 之间只能出现一个 SEP，SEP 之前的 token 数扣除 START 后应能被 $B_f = 49$ 整除，SEP 之后、END 之前的 token 数应能被 $B_e = 14$ 整除，面索引 token 必须位于预设范围内，坐标 token 必须落在量化坐标段。上述检查可以提前发现拼装错误，避免模型在训练中同时学习多种不一致格式。

格式检查还承担了调试不同预处理版本的作用。本文实现中曾保留原始采样版、量化版和服务器镜像脚本，如果三个版本在分块长度、词表偏移或特殊 token id 上不一致，训练阶段可能不会立即报错，却会在生成和重建阶段表现为大面积 S1 失败。因此，本文把词表大小、PAD id、START / SEP / END id、面段长度和边段长度都写入日志，并在训练配置中重复记录，确保数据侧与模型侧含义一致。第~5 章中 hr2\_patch1 的 0 次重建失败，也从结果层面说明最终使用的数据格式和解析器之间是匹配的。

这类检查属于工程实现细节，但对 B-rep 生成任务非常关键。自然语言模型生成一句语法不完全正确的话，仍可能被人理解，而本文序列若少一个 SEP、错一个面索引段或多出一个坐标 token，解析器就可能无法确定后续 token 属于面还是边。通过在数据入口处固定序列语法，模型学习任务被简化为在合法骨架内预测具体 token，从而降低无意义格式错误对训练和评测的干扰。

\section{拓扑感知排序方法}

\subsection{排序目标}

在固定词表与序列骨架之外，面 / 边在序列中的排列顺序也是序列化设计的一个隐性维度。它决定了相同 B-rep 在不同次预处理中是否能得到同一份 token 序列，也决定了“面 $f$”与“其邻接边 $e$”在序列中的距离。若按任意顺序排列，相邻几何要素之间的位置差完全由随机置换决定，期望距离可达数百 token，在自回归注意力下，远距离信号需要经过若干层的间接传递才能被聚合，模型对局部一致性的建模会更难，对长序列的稳定生成尤其不利。为缓解这一问题，本文为面与边各设计了一种确定性排序方案，目标有两个：第一是提高拓扑相邻元素在序列中的位置邻近性，让“先讲面、再讲边”的局部叙述结构在数据中反复出现，第二是降低同一 B-rep 在不同次预处理下的样本差异，使等价实体倾向于得到相同或相近的 token 序列。需要预先说明的是，这种确定性排序并不能完全消除参数化任意性，仅在工程上压缩了排列噪声的方差。

\begin{figure}[!htbp]
    \centering
    \includegraphics[width=1\textwidth]{插图/fig3-2_topology_sort.pdf}
    \caption{拓扑感知排序策略示意图。面排序以面—边邻接图为基础，通过深度优先遍历使拓扑相邻的面在序列中尽量靠近，边排序再结合已得到的面访问顺序，对每个面关联的边进行确定性排列，从而降低同一 B-rep 在不同预处理轮次中的序列差异。}
    \label{fig:topology-sort}
\end{figure}

\subsection{深度优先面排序}

面排序以面—边邻接图作为基础。算法从面索引最小的面出发，按深度优先方式遍历整张图：访问当前面，对其每条邻接边，找到共享这条边的另一个面，若未访问则压栈递归。最终得到一个完整的面访问序列 $\pi$。具体伪代码如\autoref{alg:dfs-face-sort} 所示。

\begin{algorithm}[H]
    \caption{DFS 面排序算法}
    \label{alg:dfs-face-sort}
    \begin{algorithmic}[1]
        \Require 面集合 $F$，边集合 $E$，面—边邻接表 $\text{adj}$
        \Ensure 有序面序列 $\pi$
        \State $\text{visited}[f] \gets \text{false}$，对所有 $f \in F$
        \State $\pi \gets [\,]$
        \State $\text{stack} \gets [\arg\min_f f.\text{id}]$
        \While{$\text{stack}$ 非空}
            \State $f \gets \text{pop}(\text{stack})$
            \If{$\text{visited}[f]$} \State \textbf{continue} \EndIf
            \State $\text{visited}[f] \gets \text{true}$
            \State 将 $f$ 追加到 $\pi$
            \For{每条 $e \in \text{adj}[f]$}
                \State 设 $f'$ 为与 $f$ 共享 $e$ 的另一面（若存在）
                \If{$f'$ 存在且未访问}
                    \State 将 $f'$ 压入 $\text{stack}$
                \EndIf
            \EndFor
        \EndWhile
        \State \Return $\pi$
    \end{algorithmic}
\end{algorithm}

DFS 保证拓扑邻接的面在序列中尽量靠近，并具有连通分量内的局部聚类性质。当 B-rep 有多个连通 Shell 时，DFS 在第一个 Shell 完成后自然继续从下一个未访问面开始，不需要额外切换逻辑。

\subsection{面索引关联边排序}

在每个面内部，边按“邻接面索引最大者”优先排序。给定面 $f_k$ 的邻接边集合 $\{e_j\}$，对每条 $e_j$ 计算

\begin{equation}
    \text{key}(e_j) = \max_{f' \ne f_k,\ f' \in \text{adj}(e_j)} \pi^{-1}(f'),
    \label{eq:edge-key}
\end{equation}

其中 $\pi^{-1}(\cdot)$ 是面在 $\pi$ 中的位置，当 $e_j$ 为自由边时取 $-\infty$。然后按 key 降序排列。具体伪代码如\autoref{alg:max-idx-edge-sort} 所示。

\begin{algorithm}[htbp]
    \caption{最大索引关联边排序算法}
    \label{alg:max-idx-edge-sort}
    \begin{algorithmic}[1]
        \Require 面 $f_k$，其邻接边集合 $\text{adj}_e[f_k]$，面排序 $\pi$
        \Ensure $f_k$ 的有序边序列
        \State $\text{keys} \gets [\,]$
        \For{每条 $e \in \text{adj}_e[f_k]$}
            \State 设 $f'$ 为与 $f_k$ 共享 $e$ 的另一面（或为空）
            \State $k \gets \pi^{-1}(f')$ 若 $f'$ 存在，否则 $-\infty$
            \State 将 $(e, k)$ 追加到 $\text{keys}$
        \EndFor
        \State 按 $k$ 降序对 $\text{keys}$ 排序
        \State \Return $[e \text{ for } (e, k) \in \text{keys}]$
    \end{algorithmic}
\end{algorithm}

该策略让“跨越较远面索引”的边出现在面内部边段的前段，使自回归生成时模型可以先看到跨度较大的边，再补齐局部边，便于在序列早期把握全局拓扑骨架。

从序列学习角度看，最大索引关联边排序的作用是把边段顺序与面访问顺序建立确定性联系。若边完全按 OpenCASCADE 遍历顺序输出，不同 STEP 文件或不同导出环境可能给出不同排列，模型会把这种排列差异也当成需要学习的随机性。采用面排序位置作为 key 后，同一类拓扑结构更容易形成相似的边段模式，减少由数据读取顺序带来的噪声。该策略尤其适合本文这种没有显式图神经网络消息传递、完全依赖序列上下文建模的架构。


\subsection{排序策略适用性说明}

本文将上述 DFS 面排序与最大索引边排序作为序列化设计的一部分提供，目的是提高局部一致性与建模便利性。本论文未单独设计“随机排序 vs 拓扑感知排序”的受控对照实验，因此不对排序策略对最终有效率的独立贡献做定量声明，相关消融实验作为未来工作列入 §5.6。

\section{序列长度与过滤准则}

排序完成后，单个样本的 token 序列长度由面数 $F$ 与边数 $E$ 决定：每个面贡献 $B_f = 49$ 个 token、每条边贡献 $B_e = 14$ 个 token，加上 3 个结构 token：

\begin{equation}
    L \approx 49 F + 14 E + 3.
    \label{eq:seq-length}
\end{equation}

在 DeepCAD 子集上常见样本的 $F$ 与 $E$ 数量级使得 $L$ 远小于 4\,096 上限。数据加载阶段会过滤超过该长度上限的样本，实际训练所读取的 \path{data/nurbs_sequences_v1_patch.pkl} 中，过滤后的最终训练集包含 64\,351 个样本，验证集包含 6\,096 个样本。

由\autoref{eq:seq-length} 可见，序列长度对面数比对边数更敏感：每增加一个面会增加 49 个 token，而每增加一条边只增加 14 个 token。这一系数差异直接影响哪些零件容易触及 4\,096 token 的上限。具有大量小面片、倒角或孔洞的零件会因为面数较多而迅速接近上限，由少数大平面和圆柱面构成的零件则即便有较多边，序列依然较短。本文采用固定上限过滤，是为了保证训练批处理稳定和注意力计算可控：一旦不同样本的长度差异过大，PAD 比例会显著上升，有效计算密度反而下降。被过滤掉的长序列样本通常对应更复杂的几何结构，如多腔体、密集小特征、装配级零件等。这意味着当前模型主要验证中等复杂度 DeepCAD 零件上的可行性，对极复杂工业零件的覆盖仍需后续扩大上下文长度（如稀疏注意力、Ring Attention）或改进压缩表示（如将面片预编码为更短的几何 token）。
