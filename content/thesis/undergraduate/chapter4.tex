\section{模型总体架构}

\subsection{窗口化输入与生成头的两种分支}

模型把第~3 章生成的分组序列张量（形状 $(L, 3)$）按窗口大小 $W$ 进一步组织为 $(L', W)$ 形式，其中 $L'$ 是窗口数。配置参数 \texttt{local\_window\_size} 控制窗口大小：当 $W = 1$ 时退化为“标准 next-token”自回归（每一行作为单步预测目标），主干 Transformer 的最终隐藏状态经 \texttt{ar\_lm\_head} 直接得到下一个 token 的 logits；当 $W > 1$ 时启用 CompoundPointDecoder，对窗口内的 $W$ 个分量做第二层因果自回归。

\textbf{本文主实验使用 $W = 1$ 配置}。训练日志明确记录 \texttt{local\_window\_size = 1}，对应标准 next-token 头（主干 norm $\to$ \texttt{ar\_lm\_head}），不启用 CompoundPointDecoder。CompoundPointDecoder 作为代码中的可选扩展保留，相关讨论见 §4.4。

采用 $W = 1$ 的直接好处是模型行为容易验证。输入序列中的每个有效 token 都对应一个明确的下一 token 预测目标，训练损失可以直接解释为 token 级交叉熵；推理时每一步只新增一个 token，生成过程与第~3 章定义的 1D 序列完全一致。对于本文这种以工程闭环为主的本科毕设，先采用最朴素、最可复查的分支更有利于定位问题：如果生成结果失败，可以从 token 类型错误、END 终止异常、反量化坐标异常、NURBS 拟合失败或拓扑缝合失败等环节逐步排查。


\subsection{级联结构}

主实验中模型由两段级联构成：

\begin{enumerate}
    \item \textbf{CompoundPointEmbedding}：把窗口内 $W$ 个 token id 融合为一个 $d_{\text{model}}$ 维向量；
    \item \textbf{LLaMA3 风格主干}：$N$ 层因果 Transformer，对窗口向量序列做长程依赖建模；其末层经 RMSNorm 后接 \texttt{ar\_lm\_head}，直接输出下一个 token 的 logits。
\end{enumerate}

从数据流上看，输入 token 首先被映射到连续向量空间，随后由因果 Transformer 在时间维上聚合历史上下文，最后再投影回离散词表。该流程与语言模型生成文字的基本范式一致，但本文 token 的语义更混合：同一词表中既有几何坐标，又有面索引和结构控制符。因此，模型不仅要学习坐标分布，还要学习每个位置应该输出哪类 token，以及 START、SEP、END 等符号的出现时机。第~3 章中稳定的序列骨架为这种学习提供了强先验。

主干隐藏维度 $d_{\text{model}}=256$ 相比大语言模型明显更小，这是由数据规模、序列长度和硬件预算共同决定的。CAD token 序列虽然具有长程依赖，但训练样本规模远小于互联网文本，过大的模型可能在有限数据上过拟合，也会显著增加 4\,096 长度下的显存消耗。本文采用 8 层、256 维的轻量配置，目标是让模型足以捕获 DeepCAD 子集中的主要结构规律，同时保证 4 张 H20 上可以完成 200 epoch 训练。

\section{CompoundPointEmbedding：窗口向量化}

\subsection{Token Embedding 与线性融合}

CompoundPointEmbedding 包含一张大小为 $\text{vocab\_size} \times d_{\text{model}}$ 的 token Embedding 表 $E_{\text{val}}$ 与一层线性融合 $W_{\text{fuse}} \in \mathbb{R}^{(W \cdot d_{\text{model}}) \times d_{\text{model}}}$。给定窗口 $x_t \in \mathbb{Z}^{W}$，先按位查表得到 $W$ 个 $d_{\text{model}}$ 维向量并展平，再通过线性层投影回 $d_{\text{model}}$：

\begin{equation}
    \mathbf{h}_t = W_{\text{fuse}} \cdot [E_{\text{val}}(x_t^{(0)});\, E_{\text{val}}(x_t^{(1)});\, \ldots;\, E_{\text{val}}(x_t^{(W-1)})].
    \label{eq:embed-fuse}
\end{equation}

在主实验 $W = 1$ 的设定下，融合层退化为单 token 的线性变换。当 $W = 3$ 时该结构可同时将一个三元组的三个分量打包成一个窗口向量，避免使用三张独立的 Embedding 表。

即使在 $W = 1$ 时，CompoundPointEmbedding 仍然承担两个作用。首先，它把离散 token id 映射到连续空间，使坐标 token、面索引 token 和结构 token 都可以被 Transformer 统一处理。其次，线性融合层保留了与窗口化分支兼容的接口，便于在不重写主干代码的情况下切换不同窗口大小。这样的实现方式牺牲了一点结构极简性，但提高了代码复用性，也使本文实验与后续扩展能够共享同一模型入口。


\subsection{词表与 PAD}

主实验中 \texttt{vocab\_size = 4054}（数据词表 4053 + 1 个 PAD），PAD 的 id 为 4053。模型保留 PAD 在 Embedding 表中的 id 以便填充位置可参与查表，但训练时这些位置的 label 设为 $-100$，由交叉熵损失忽略（详 §4.5）。

\section{LLaMA3 风格主干}

\subsection{块结构}

主干由 $n_{\text{layers}} = 8$ 个同构 LLaMA3 风格 Transformer block 串联。每个 block 采用 pre-norm 残差结构：

\begin{align}
    \mathbf{h}' &= \mathbf{h} + \text{GQA}\big(\text{RMSNorm}(\mathbf{h})\big), \label{eq:block-attn} \\
    \mathbf{h}'' &= \mathbf{h}' + \text{SwiGLU}\big(\text{RMSNorm}(\mathbf{h}')\big). \label{eq:block-ffn}
\end{align}

注意力部分采用 Grouped Query Attention，前馈部分采用 SwiGLU，残差前后各使用一层 RMSNorm。本文主干配置如下：层数 $n_{\text{layers}} = 8$、查询头数 $n_{\text{heads}} = 8$、KV 头数 $n_{\text{kv\_heads}} = 4$（每个 KV 头由 2 个 Q 头共享）、隐藏维度 $d_{\text{model}} = 256$、前馈中间维度 $\text{dim\_ffn} = 1024$，RoPE 的 $\theta = 500\,000$，RMSNorm 的 $\epsilon = 10^{-5}$。

pre-norm 结构的优点是深层训练更稳定。每个子层在进入注意力或前馈网络之前先做归一化，残差分支则保留原始隐藏状态，有助于梯度在多层网络中传播。虽然本文主干只有 8 层，并不属于极深网络，但序列长度较长、token 类型混合，训练稳定性仍然重要。RMSNorm 相较 LayerNorm 计算更简单，也与 LLaMA 系列结构保持一致。

\subsection{RoPE 应用细节}

RoPE 仅对 Q、K 应用，不作用于 V。RoPE 缓存的最大序列长度由配置中的 \texttt{max\_seq\_len} 决定，并在遇到超长输入时自动扩展。RoPE 的相对位置不变性使得主干在窗口数较多的场景下仍可保持稳定的注意力点积。

\subsection{KV cache 与训练加速}

主干在推理阶段使用 KV cache：每层缓存最近一次前向的 $K$、$V$，新增窗口只需与缓存做注意力点积，把单步推理复杂度由 $O(T^2)$ 降至 $O(T)$。训练阶段启用梯度检查点（gradient checkpointing）与 PyTorch AMP 混合精度（\texttt{use\_amp = True}），在 H20 上以 FP16/BF16 算子加速矩阵乘累加，训练日志中记录了“梯度检查点已启用（节省显存）”。

\subsection{主干末端：RMSNorm + ar\_lm\_head}

主干末端为一层 RMSNorm，其输出的隐藏状态 $\mathbf{h}_t \in \mathbb{R}^{d_{\text{model}}}$ 直接送入 \texttt{ar\_lm\_head}（一层无偏置 \texttt{nn.Linear}）映射到 $\text{vocab\_size}$ 维 logits，预测下一个 token 的分布。该“主干 norm $\to$ ar\_lm\_head”即本文主实验所采用的标准 next-token 头。

由于输出词表同时包含面索引、坐标和结构 token，\texttt{ar\_lm\_head} 的每一次预测都在完整词表上竞争。训练过程中，模型会根据上下文自动学习不同位置的 token 类型概率。例如，在 START 之后的早期位置更可能输出面段坐标；在面段达到某种长度模式后更可能输出 SEP；在边段完成后更可能输出 END。这种隐式语法学习使实现保持简洁，但也意味着模型偶尔可能打破序列格式。第~5 章中 S1 样本的减少，间接反映了 hr2\_patch1 在生成可解析序列方面已有较好稳定性。

\section{未启用的窗口化扩展说明}

\begin{sloppypar}
本文代码中保留了面向 $W > 1$ 窗口化分支的 \texttt{CompoundPointDecoder} 与 \texttt{LocalBlock} 实现。其原始设计意图是在一个窗口包含多个语义分量时，将主干隐藏状态作为上下文，再对窗口内部 token 做局部因果分解。但本文主实验配置为 \texttt{local\_window\_size = 1}，模型实例化时 \texttt{LLaMA3ARModel} 选择“\texttt{self.local\_decoder = None}、\texttt{self.ar\_lm\_head = nn.Linear(...)}”分支，\textbf{完全不构建也不调用 CompoundPointDecoder}。因此，本文报告的训练损失、生成有效率和重建结果均来自标准 next-token 分支，不依赖该扩展模块。
\end{sloppypar}

为避免将未启用模块误写为本文核心方法，本章不再单独给出 CompoundPointDecoder 结构图，也不对其效果作定量声明。后续若启用 $W > 1$ 的窗口化设定，可在相同数据划分与评测口径下补充对照实验，再讨论该模块对生成质量和有效率的影响；相关方向列入 §5.6 后续工作。

\section{损失函数与训练策略}

\subsection{交叉熵损失}

总损失为 $-100$ 屏蔽下的 token 级交叉熵：

\begin{equation}
    \mathcal{L} = -\sum_{t,\, \text{label} \ne -100} \log p_{\theta}(y_t \mid y_{<t}),
    \label{eq:ce-loss}
\end{equation}

实现使用 \texttt{nn.CrossEntropyLoss(ignore\_index = -100)}。PAD 位置的 label 在数据侧已置为 $-100$，因此不会产生梯度。

交叉熵损失是训练阶段最直接的优化目标，但它与最终 BRepCheck 通过率并非完全等价。较低的 token 预测损失通常意味着模型更接近训练数据分布，生成序列也更可能满足常见骨架；然而，一个 token 的局部错误可能对最终几何产生不同影响：坐标 token 的小偏差可能只带来轻微几何误差，SEP 或面索引 token 的错误却可能导致整个解析失败。因此，本文在第~5 章同时报告验证集 CE、PPL 和生成有效率，避免只用损失曲线推断工程可用性。

\subsection{优化器与学习率}

优化器为 AdamW\cite{kingma_2015_adam,loshchilov_2019_adamw}，初始学习率 $5 \times 10^{-4}$、权重衰减 $0.01$、dropout $0.1$，学习率调度采用“warmup $\to$ 中段稳定 $\to$ 末段衰减”三段式形态，对应曲线共有 201 条 step-level 记录。训练全程启用混合精度\cite{micikevicius_2018_mixed_precision}（\texttt{use\_amp = True}），并在主干上启用梯度检查点以节省显存。

AdamW 将权重衰减与自适应梯度更新解耦，适合 Transformer 类模型训练。本文的学习率相对常见大语言模型预训练不算低，但模型规模较小、数据集经过规则化处理，因此能够在 200 epoch 内持续下降。dropout 用于抑制有限数据上的过拟合，梯度检查点则通过在反向传播时重算部分中间激活换取显存节省。混合精度训练降低矩阵乘法开销，但对几何生成任务而言仍需关注数值稳定性；本文训练日志未观察到损失爆炸或 NaN，说明该配置在当前硬件与数据上可用。

\subsection{分布式训练}

训练使用 PyTorch DDP（\texttt{torchrun --nproc\_per\_node=4}），4 张 H20 各占 1 进程，每进程 batch size 为 32，全局 batch size 为 128。详细超参数与硬件配置见 §5.1。

\section{推理生成流程}

\subsection{自回归采样}

推理时输入只含一个 START 三元组，模型按因果顺序逐窗口生成；每一步从主干末端的 \texttt{ar\_lm\_head} 取 logits 后做温度缩放，再依次按 top-k 与 top-p 过滤，最后从过滤后的分布中按 \texttt{torch.multinomial} 采样。本文实际使用的采样配置为 top-k = 1 + top-p = 0.9（贪心式采样并保留累积概率 90\% 的候选集合），该类截断采样思路与神经文本生成中的 nucleus sampling 口径相关\cite{holtzman_2020_nucleus}，生成目录命名为 \texttt{1000\_1\_0.9}。

伪代码如\autoref{alg:ar-generate} 所示。

\begin{algorithm}[htbp]
    \caption{自回归序列生成（$W = 1$ 主干 next-token 分支）}
    \label{alg:ar-generate}
    \begin{algorithmic}[1]
        \Require 训练好的模型 $M$，最大窗口数 $L_{\max}$
        \Ensure token 序列 $x$
        \State $x \gets [\text{START}]$，KV cache $\gets \varnothing$
        \While{窗口数 $< L_{\max}$}
            \State $\mathbf{h} \gets M.\text{backbone}(x_{\text{last}};\ \text{KV cache})$
            \State $\text{logits} \gets M.\text{ar\_lm\_head}(\mathbf{h})$
            \State $t \gets \text{sample}(\text{logits};\ T, k, p)$
            \State 将 $t$ 追加到 $x$
            \If{$t = \text{END}$} \State \textbf{break} \EndIf
        \EndWhile
        \State \Return $x$
    \end{algorithmic}
\end{algorithm}

当模型输出 END 时立即终止；如未输出 END 而达到最大窗口数则强制截断。

推理阶段还面临长度控制问题。若模型过早输出 END，序列可能只包含不完整面段或缺少边段；若迟迟不输出 END，则序列会达到 \texttt{max\_seq\_len} 后被截断，解析时也容易失败。本文把这两类情况都交由三态统计反映，而不是在采样阶段手写复杂规则强制补全。这样可以更真实地评估模型本身学到序列边界的程度。

\subsection{特殊 token / 索引 token 的位置约束}

由于本文使用 $W = 1$ 的标准 next-token 头，原始 token 序列里的“面索引、坐标、特殊符号”三类 token 由模型按位置依次生成，序列骨架（face\_block / edge\_block 的分隔位置、SEP 的出现位置等）由训练数据中的高频统计模式自然学到。

\section{逆向重建：序列 $\to$ NURBS $\to$ B-rep $\to$ STEP / STL}

逆向重建管线由序列解析、NURBS 拟合、拓扑缝合与几何有效性检查组成，整个流程分为四步。

\begin{figure}[!htbp]
    \centering
    \includegraphics[width=0.82\textwidth]{插图/fig4-4_reconstruction_pipeline.pdf}
    \caption{逆向重建管线示意图。生成序列首先被解析为面段与边段，并通过反量化恢复为三维采样点；随后分别拟合 NURBS 曲面和曲线，依据面索引关系完成拓扑缝合与实体构建，最后导出 STEP / STL 文件并调用 OpenCASCADE 的 BRepCheck 进行有效性检查。}
    \label{fig:reconstruction-pipeline}
\end{figure}

\subsection{Token 序列解析与反量化}

解析器从 START 起按 SEP 切分面段与边段，再按 face\_block = 49 与 edge\_block = 14 切分每个面与每条边。坐标 token 由反量化函数

\begin{equation}
    x = \frac{2 (t - \text{offset})}{N_q} - 1
    \label{eq:dequantization}
\end{equation}

恢复为 $[-1, 1]$ 内的浮点坐标。面索引段直接读取 token 数值作为面索引；特殊符号段触发段落切换或终止。

\subsection{NURBS 曲面 / 曲线拟合}

对每个面，将 16 个反量化点送入 \texttt{GeomAPI\_PointsToBSplineSurface}，阶数取 $p = q = 3$，节点向量自动均匀分布，得到 NURBS 曲面；对每条边，将 4 个反量化点送入 \texttt{GeomAPI\_PointsToBSpline}，阶数 3，得到 NURBS 曲线。拟合失败（如点共线、奇异参数）会抛 \texttt{StdFail\_NotDone} 异常，归入三态记录中的 S1（未产出）。

\subsection{拓扑缝合与实体构建}

\begin{sloppypar}
对每个面创建 \texttt{BRepBuilderAPI\_MakeFace}，再对面集合使用 \texttt{BRepBuilderAPI\_Sewing} 缝合成 Shell；使用 \texttt{ShapeFix\_Shape} 修复容差不闭合的边；最后用 \texttt{BRepBuilderAPI\_MakeSolid} 封闭为 Solid。Solid 经 OCC 的 STEP writer 导出 \texttt{.step} 文件，并同时产出一份三角化的 \texttt{.stl} 便于可视化。
\end{sloppypar}

该阶段是生成式 CAD 与一般三维点云生成区别最明显的部分。点云或网格生成只需产出一组点或三角面即可展示，而 B-rep 重建必须让曲面、曲线、边界和体拓扑在内核中形成一致结构。Sewing 操作尝试把相邻面片按容差缝合，ShapeFix 则对部分可修复的容差问题进行调整；但这些工具不能把任意错误序列变成合法实体。如果生成的邻接面索引混乱、边曲线远离相邻面，或面片本身退化严重，最终仍会在构造 Solid 或 BRepCheck 阶段失败。

因此，逆向重建既是输出转换模块，也是对前面生成质量的严格过滤器。本文没有把 ShapeFix 的自动修复能力视为模型性能的一部分来夸大，而是将最终 OCC 判定结果作为统一口径：能写出 STEP 且通过 BRepCheck 才进入 S3；写出但无效进入 S2；无法写出进入 S1。这样的统计方式可以清楚地区分“模型生成了结构上较完整的序列”和“后处理勉强写出了文件但仍不合法”两种情况。

\subsection{几何有效性检查}

最终调用 \texttt{BRepCheck\_Analyzer(solid).IsValid()} 作为判据，结果写入评测 JSON。有效性检查提供两个变体：一个从 STEP 路径读入并校验，另一个直接接收 \texttt{TopoDS\_Shape} 实例进行校验。判据语义与第~2 章 §2.5 一致：\texttt{IsValid()=True} 即认定该样本“有效”。

\subsection{三态记录}

评测脚本输出五个标量：序列总数、重建成功、重建失败、有效 BREP、无效 BREP，按第~2 章\autoref{tab:three-state-def} 的三态口径分别对应 S1（重建失败）、S2（无效 BREP）、S3（有效 BREP）三个统计槽位。该输出格式直接支撑第~5 章主表口径。

五标量输出虽然简单，但对论文复核非常关键。只报告有效率百分比容易掩盖分母差异：例如，1000 次采样中 500 次无法写出 STEP、剩余 500 次全有效，与 1000 次采样全部写出且 500 次有效，在 of saved 口径下差异很大。本文同时记录采样次数、已写出 STEP 数和 BRepCheck Valid 数，可以让读者判断有效率来源。第~5 章对早期 hr2 基线和 hr2\_patch1 的比较也正依赖这一点：两者不仅 of saved 有效率不同，S1 重建失败比例也明显不同。
