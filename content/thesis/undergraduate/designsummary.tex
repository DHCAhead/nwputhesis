%% 毕业设计小结

\begin{designsummary}
本毕业设计以“基于深度学习的 B-rep 端到端生成算法研究”为题，目标是在不依赖显式 VQ-VAE 码本的前提下，验证一种“点级 NURBS 序列化 + 标准 next-token 自回归 Transformer”的端到端 B-rep 生成框架，并把整条管线落到可被 OpenCASCADE 校验的 STEP 输出上。整个工作可以归纳为研究、实现、实验与写作四个阶段。

研究阶段，我系统阅读了 PolyGen、DeepCAD、SkexGen、BrepGen、SolidGen 等代表性工作，把现有 B-rep 相关生成方法按“中间表征”维度梳理为 CAD 操作序列、草图—拉伸解耦码本、结构化潜空间扩散与无码本 B-rep 自回归四条路线，比较其优势与限制。在此基础上，本课题被定位为“点级量化 + 自回归 Transformer + OpenCASCADE 逆向重建”的工程化探索——它选择放弃码本压缩与潜空间扩散，把几何精度上界直接绑在标量量化粒度上。本阶段同步完成了 B-rep 拓扑层级、NURBS 数学定义、Transformer / LLaMA3 关键组件、OpenCASCADE BRepCheck 评测口径等技术铺垫。

实现阶段，我完成了五个环节的全部代码实现：数据预处理（STEP $\to$ 拓扑分解 $\to$ NURBS 采样 $\to$ 4000 级标量量化 $\to$ DFS 面排序 + 最大索引边排序），自回归生成模型（LLaMA3 风格主干，由 8 层 GQA、SwiGLU、RMSNorm、RoPE 组成，标准 next-token 头，CompoundPointDecoder 作为可选扩展但本次主实验未启用），训练流程（PyTorch Lightning + DDP + AMP + 梯度检查点），推理（top-k = 1 与 top-p = 0.9 的保守自回归采样），逆向重建（GeomAPI NURBS 拟合 + BRepBuilderAPI Sewing + ShapeFix + BRepCheck 校验）。数据词表规模 4\,053（50 面索引 + 4\,000 量化坐标 + 3 个结构 token），模型 \texttt{vocab\_size = 4\,054}（额外引入 PAD 用于批内对齐），face\_block = 49（16 个三维点 + 1 个面索引），edge\_block = 14（2 个面索引 + 4 个三维点）。

实验阶段，我在 DeepCAD 去重子集（训练 64\,351 / 验证 6\,096）上完整训练了 200 个 epoch（4 张 NVIDIA H20、端到端 17 小时 28 分钟）。最终模型 hr2\_patch1 验证集交叉熵损失收敛至 0.10486（PPL $\approx$ 1.110），1\,005 个独立生成样本经反向重建后，全部成功写出 STEP、BRepCheck 通过率为 97.51\%（of saved 与 of attempts 均为 97.51\%），早期 hr2 基线在 3\,000 个写出样本上的对应数字为 90.40\%。两者均显著超过毕业设计任务书 60\% 的红线指标。同时记录了 S1（未产出）/ S2（产出但无效）/ S3（产出且有效）三态分布，并明确指出本文未单独对拓扑感知排序与 CompoundPointDecoder 等组件做受控对照消融，因此不对其独立贡献做定量声明。

写作阶段，按照“研究问题 $\to$ 相关技术 $\to$ 序列化与预处理 $\to$ 自回归模型与重建 $\to$ 实验与总结”五章结构组织全文。写作过程中严格遵守“每一个关键技术主张必须可追溯到文献、代码、实验日志或学校任务书”的证据规范——对于尚未在本论文中完成的实验（跨数据集泛化、排序受控消融、CompoundPointDecoder 启用对照、几何相似度评估、VQ-VAE 路线复现等），一律放入论文末尾的局限性与后续工作部分，宁缺勿造，对于实测有效率、训练时长、硬件配置、数据规模等关键数字，全部对照训练日志和评测脚本输出做一次以上交叉核对，避免因模糊表达造成数字错配。

通过本毕业设计，我在 CAD 几何表征、深度生成模型工程实现、大规模训练实验组织、学术写作与证据管理等方面均获得了系统性训练，也对自己在 B-rep 几何精度、跨数据集泛化、几何相似度指标体系等方向的不足有了较为清楚的认识——这些不足已经以条目形式在第~5 章末尾以“后续工作”的方式列出，便于继续推进。
\end{designsummary}
