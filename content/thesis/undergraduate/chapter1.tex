\section{研究背景与意义}

\subsection{CAD 与 B-rep 的工程基础}

计算机辅助设计（CAD）已成为机械、航空、消费电子等行业产品研发的核心环节。在 CAD 系统内部，边界表示是描述实体几何最常见的形式：一个实体由若干 Shell 组成，每个 Shell 由有方向的 Face 组成，Face 由若干 Loop 围成的边界限定，每个 Loop 又由 Edge 序列组成，Edge 端点为 Vertex\cite{requicha_1980_representations,mantyla_1988_solid_modeling,hoffmann_1989_geometric_solid_modeling,piegl_tiller_1997_nurbs}。这种“几何 + 拓扑”分离的层级结构使得参数化建模、布尔运算和工艺仿真都可以在统一的数据结构上进行。STEP（ISO 10303）与 IGES 等工业交换格式都直接序列化 B-rep 数据，行业内主流 CAD 内核（如 Parasolid、ACIS 与开源的 OpenCASCADE）也均以 B-rep 作为内部表示，近年的开放 CAD 内核与 B-rep/CSG 转换研究也说明，几何内核层面的可交换、可校验表示仍是智能 CAD 系统落地的重要基础\cite{liang_2023_cad_kernel,tai_2023_brep_csg}。

相比之下，离散几何表示（如三角网格 STL、体素或纯点云）虽然便于可视化和有限元前处理，但都丢弃了 B-rep 中“曲面参数 + 拓扑层级”这两类信息：网格只保留三角片的顶点坐标和拓扑邻接，缺少裁剪曲线和参数域，体素以体素格的占据状态近似实体内外，分辨率受立方级开销限制，点云只表达空间采样分布，连面与边的概念都没有。任何面向工业 CAD 工作流的生成模型若最终只能输出这类离散表示，都需要额外的等值面提取、曲面拟合或表面重建步骤才能勉强回到 B-rep，整个过程通常会丢失尖锐特征、引入额外的拓扑修复成本，难以直接对接尺寸编辑、布尔运算与 CAM 等下游环节。因此，本文在生成对象的选择上直接以“可被 OpenCASCADE 等几何内核读取并校验的 B-rep 实体”为终点，而不是先生成网格再做事后转换。

从工程角度回看，B-rep 长期主导 CAD 内核的核心位置，根本原因并不只是它能描述复杂曲面，而是它在同一份数据结构中同时表达了“形状是什么”和“各几何要素如何连接”。机械零件能否被实际使用，往往取决于若干非视觉特征：边界是否完整闭合、面片取向是否一致、共享边是否被相邻面正确引用、曲面参数域是否与其裁剪曲线匹配。三角网格可以近似零件的外观但难以保留设计意图，点云能够表达空间采样分布却缺少拓扑边界，隐式场能够形成连续表面但仍需通过等值面提取或曲面拟合转回可编辑实体，每一步转换都会引入或放大错误。生成模型若希望真正服务于 CAD 建模、工程分析和制造准备，就不能只追求视觉上相似的三维形状，还需要直接产出可被几何内核读取、修复和校验的实体结构。这一定位也是本文区别于多数三维生成工作的核心立场。


\subsection{深度学习三维生成研究进展}

近年来生成式人工智能在二维图像和自然语言任务上取得显著进展，三维生成方向也已经从早期的体素表示与 ShapeNet 等通用三维数据集出发，逐步扩展到单 / 多视图重建、点云生成、参数化曲面、隐式表面与多边形网格的自回归生成\cite{wu_2015_3dshapenets,chang_2015_shapenet,choy_2016_3dr2n2,fan_2017_pointset,qi_2017_pointnet,qi_2017_pointnetplusplus,groueix_2018_atlasnet,achlioptas_2018_learning_representations,park_2019_deepsdf,mescheder_2019_occupancy,yang_2019_pointflow,nash_2020_polygen}。面向 CAD 场景的数据驱动智能建模已成为一个活跃方向\cite{tan_2025_intelligent_cad_review}，但能够直接产出工业可用 B-rep 的方法仍然不多，且大部分集中在草图序列、参数化操作或离散码本的间接表示路线，几何精度与拓扑闭合性仍存在改进空间。

从上述趋势可以看出，CAD 生成研究与一般三维生成存在两个关键差别。第一个差别在于建模目标：自然图像或网格生成只需把单个连续分布学好，而 B-rep 生成同时面对一组结构化几何要素（面、边、顶点）和它们之间的离散拓扑关系，要求模型既能产生几何采样点的连续坐标，又能产生面索引、特殊符号等离散结构 token。大模型式的自回归建模已经在文本、代码、蛋白质序列等结构化对象上证明，复杂分布可以被组织为离散 token 序列，由 decoder-only Transformer 通过 next-token 预测捕获其中的长期依赖，因此，只要能把 B-rep 的面、边和几何采样点稳定地拼成 token 序列，就有可能复用这套成熟的生成范式来表达“前文约束后文”的几何—拓扑生成关系。第二个差别在于表征压缩的代价：三维几何具有强结构性，一旦把整个 B-rep 压成单一全局潜变量或离散码本编号，局部拓扑细节就被压成几个数字，重建阶段必须靠解码器把它们“扩”回来，模型对几何精度的可见控制点也随之减少。本文之所以选择在数据表示阶段保留面、边和邻接面索引等局部结构、仅在坐标分量上做标量量化，正是希望让几何精度上界保持在标量量化粒度上，而不是被码本规模或潜变量分布的拟合质量隐式封顶。


\subsection{研究意义}

直接生成可用的 B-rep 模型，对工业设计具有三方面较具体的实际价值。其一是降低参数化建模的操作门槛：当前 CAD 软件的草图—约束—拉伸—布尔工作流需要用户掌握特征建模思维，对非专业人员而言学习曲线陡峭，若生成模型可以由稀疏意图（少量约束、关键尺寸或语义描述）直接给出候选 B-rep 实体，用户只需对结果做局部编辑就能继续推进。其二是缩短概念设计阶段的方案搜索时间：传统方案设计依赖设计师反复手绘、建模、评估，候选数量受人力限制，自动生成大量结构合法的候选实体后，工程师可以把精力集中在筛选与微调而非从零搭建。其三是为约束细化、拓扑优化、可制造性分析等下游算法提供初始候选——这些算法多以已有 B-rep 为输入进行迭代，“是否能拿到一个起点足够好的实体”通常是它们能否进入工程流程的关键瓶颈。本研究并不试图同时回应上述全部需求，而是把毕业设计任务书要求作为锚点，将研究目标限定为“验证一种端到端 B-rep 生成框架的工程可行性”，并以 OpenCASCADE BRepCheck 通过率不低于 60\% 作为可量化的工程指标。具体的实测结果详见第~5 章，本节先把研究问题定位在可被几何内核接受的实体级输出上。

在“工程可行性验证”这一定位下，本文并不追求一次性解决生成式 CAD 的全部问题，而是聚焦于一个较清晰的闭环：从公开 CAD 数据集中读取 B-rep，抽取面和边上的 NURBS 采样点，将其转换为可训练 token 序列，然后训练标准自回归模型生成新的 token 序列，最后通过反量化、NURBS 拟合、拓扑缝合和 BRepCheck 校验，把生成序列重新落回 STEP / STL 文件。该闭环若能稳定运行并达到任务书要求的有效率，就说明点级表示具有继续扩展的基础。在此目标下，本文后续章节将更强调数据表示、模型实现、逆向重建和评测口径的一致性，而不是对外观美观度或多样性做过度展开，前者关乎“系统是否能跑通”，后者属于后续阶段的优化方向。

\section{B-rep 生成问题界定}

\subsection{B-rep 表示及 NURBS 几何基础}

B-rep 实体的拓扑层级可由如下偏序集刻画：Solid $\to$ Shell $\to$ Face $\to$ Loop $\to$ Edge $\to$ Vertex。每个 Face 关联一个参数化曲面 $S(u, v)$，每个 Edge 关联一条参数化曲线 $C(u)$。本文采用工业中应用最广的 NURBS 曲面与曲线作为承载几何\cite{piegl_tiller_1997_nurbs}。NURBS 曲面定义为

\begin{equation}
    S(u, v) = \frac{\sum_{i=0}^{m}\sum_{j=0}^{n} N_{i,p}(u) N_{j,q}(v) w_{ij} \mathbf{P}_{ij}}{\sum_{i=0}^{m}\sum_{j=0}^{n} N_{i,p}(u) N_{j,q}(v) w_{ij}},
    \label{eq:nurbs-surface}
\end{equation}

其中 $N_{i,p}$ 与 $N_{j,q}$ 是节点向量上由 Cox--de Boor 递推得到的 B 样条基函数，$\mathbf{P}_{ij}$ 是控制点，$w_{ij}$ 是对应权重。NURBS 曲线由相应的一维形式定义。NURBS 表示之所以被多数 CAD 内核采用，主要原因有三：它具有平移和旋转不变性，使得 CAD 操作不会因为坐标系变化而需要重新参数化，它具有局部支撑性，单个控制点只影响相邻参数区间，便于工程师对局部细节做无副作用的修改，它能够通过引入权重精确表达包括圆、椭圆、二次曲面等常见工业曲线和曲面\cite{shi_2013_cagd_nurbs,piegl_tiller_2010_nurbs_cn,farin_2002_curves_surfaces}，避免了多种几何要素需要多种独立表示的混乱。本文在数据预处理阶段直接沿用 OpenCASCADE 内部的 NURBS 表示来抽取面与边的采样点，避免在表征转换阶段再引入额外几何误差。

\subsection{生成任务形式化描述}

本文考虑的是无条件 B-rep 生成任务。模型接收起始标志 START 后，自回归地输出一条 token 序列，序列经反向解析与曲面 / 曲线拟合，最终交付一个可写入 STEP 文件的 Solid。本文不在生成阶段提供任何文本或图像条件信号，目的是先验证基础的“几何 + 拓扑”联合生成能力。逆向重建阶段由序列解析、曲面 / 边拟合、拓扑缝合和 STEP 导出四个环节组成，具体流程在第~4 章展开说明。

更形式化地说，设训练样本为 $x = (x_1, x_2, \ldots, x_T)$，其中每个 $x_t$ 是统一词表中的一个 token，既可能表示量化后的坐标分量，也可能表示面索引或 START、SEP、END 等结构符号。自回归模型需要学习条件分布

\begin{equation}
    p(x) = \prod_{t=1}^{T} p(x_t \mid x_{<t}; \theta),
    \label{eq:brep-ar-factorization}
\end{equation}

并在推理阶段从 START 开始逐步采样，直到生成 END 或达到最大序列长度。与图像扩散模型从噪声场逐步去噪不同，本文的生成过程按 token 顺序展开，因此序列化顺序会直接影响模型看到的上下文结构。第~3 章设计的 face\_block、edge\_block 与拓扑感知排序，正是为了让序列中相邻 token 尽可能对应 B-rep 中具有局部关联的几何要素。

\subsection{生成任务的关键约束}

本文任务的难点首先来自几何与拓扑之间的强耦合。对二维图像或普通点云而言，局部像素或局部点集出现小幅误差，通常只会造成视觉质量下降，而在 B-rep 生成中，一个面索引错误、一条共享边缺失或一个边界方向不一致，都可能导致整个实体无法封闭。换言之，模型不仅要生成“看起来像零件”的坐标分布，还要在较长序列中维持面、边、索引和特殊符号之间的组合关系。这种组合关系具有明显的离散语法特征，但其合法性又必须通过连续几何拟合和 CAD 内核检查来最终确认，因此比单纯的离散序列生成更容易出现错误传播——一个坐标 token 的微小偏差只会让某条边略微偏离原位置，而一个面索引 token 的偏差则可能让一条边引用到不存在的面，使后续的 Sewing 与 BRepCheck 步骤直接失败。

第二个难点来自表示的非唯一性。同一 B-rep 实体在不同 CAD 内核、不同导出工具或不同遍历顺序下，可能拥有不同的 Face / Edge 编号、不同的参数域方向以及不同的曲线起点，这些差异在视觉与几何意义上完全等价，但作为离散 token 序列却是不同的样本。若直接把 OpenCASCADE 遍历得到的对象顺序作为训练序列，模型就需要额外学习这种“与几何形状无关的排列噪声”，相当于在有限数据上同时学习几何分布和置换分布。本文在预处理阶段引入面 / 边的确定性排序、UV 方向规范化和边方向统一，目的并不是宣称获得唯一的数学标准形，而是在工程上尽量减少这种等价表示带来的样本差异，使模型学习重点回到几何形态和拓扑邻接本身。

再次，本文选择点级量化而非显式码本压缩，也带来了序列长度方面的约束。每个面贡献 49 个 token，每条边贡献 14 个 token，零件稍微复杂便会产生数百到数千长度的序列，在自回归模型中，长序列不仅增加训练显存与推理时间，也使远距离依赖更难捕获。本文将最大序列长度设为 4\,096，并过滤超长样本，是在当前硬件条件和本科毕设周期下做出的折中。该设定保证了训练过程可复现，但也意味着本文更偏向验证中等复杂度零件上的可行性，对更复杂工业零件的处理仍需更长上下文或更强压缩策略。

\subsection{几何拓扑有效性判据}

本文以 OpenCASCADE 的 \texttt{BRepCheck\_Analyzer} 作为最终判据：当且仅当 \texttt{BRepCheck\_Analyzer.IsValid()=True} 时，生成实体被认定为“有效”。选择该判据有两个工程考虑。一是它直接来自将被本文用作几何内核的 OpenCASCADE 本身，其检查的对象与后续 Sewing、ShapeFix、STEP writer 共享同一份 TopoDS\_Shape，避免了“用另一份内核检查另一份内核生成的实体”所引入的判据不一致，二是它一次性覆盖了几何细节与拓扑约束两类问题：曲面拟合质量、曲线连续性属于几何层面，Wire 是否闭合、Shell 是否含游离边、是否能封闭为唯一 Solid 属于拓扑层面，二者在 \texttt{IsValid()} 中按统一逻辑判定。在统计层面，每一条采样得到的 token 序列在重建管线中被记录为以下三种状态之一，如\autoref{tab:three-state-def} 所示。

\begin{table}[!h]
    \caption{生成样本的三态记录口径}
    \label{tab:three-state-def}
    \begin{tabularx}{\textwidth}{p{2.5cm}LL}
        \toprule
        状态 & 触发条件 & 统计槽位 \\
        \midrule
        S1：未产出 STEP/STL & 解析或重建管线抛异常 / 返回 None，没有任何文件写出 & 计入“重建失败” \\
        S2：产出但 BRepCheck 无效 & \makecell[l]{写出 STEP/STL，但\\ \texttt{BRepCheck\_Analyzer}\\ \texttt{.IsValid()=False}} & 计入“无效 BREP” \\
        S3：产出且有效 & \makecell[l]{写出 STEP/STL 且\\ \texttt{BRepCheck\_Analyzer}\\ \texttt{.IsValid()=True}} & 计入“有效 BREP” \\
        \bottomrule
    \end{tabularx}
\end{table}

记 S1、S2、S3 三类样本数之和为序列总数，本文主表口径采用“有效率（of saved）$= \mathrm{S3} / (\mathrm{S2} + \mathrm{S3})$”，即在所有成功写出 STEP 文件的样本中 BRepCheck 通过的比例。

这种口径的意义在于把生成链路中的两个失败层次区分开来。S1 反映的是序列解析、曲面拟合或拓扑构造阶段是否能够产出文件，S2 反映的是文件虽然已经写出，但经 OCC 几何内核检查后仍存在不合法结构，S3 才表示完整通过逆向重建与有效性检查。对于生成式 CAD 任务，仅报告“采样了多少条序列”不足以说明工程可用性，仅报告“可视化是否像零件”也无法替代内核级检查。因此本文在第~5 章同时给出 S1、S2、S3 三态分布和 of saved / of attempts 两种口径，主结论以 of saved 为准，并在必要处说明重建失败样本对整体尝试次数的影响。

\section{国内外研究现状}

\subsection{离散与隐式三维表示方法}

针对一般三维形状的生成研究已经形成多条路线，但其表征都和 B-rep 之间存在结构性差距。体素方法以三维网格作为载体，分辨率每提高一倍计算与显存开销就增长八倍，难以扩展到工业 CAD 常见的尺寸跨度，点云方法（如 PointNet 家族）以无序点集作为输入或输出，本身轻量、对采样数变化鲁棒，但只保留了空间分布，丢弃了面、边等拓扑结构，且常用于分类与分割而非生成。以 DeepSDF、Occupancy Networks 等为代表的隐式表面方法用神经网络隐式表示符号距离场或占据概率场，再通过 Marching Cubes 等等值面提取算法生成三角网格\cite{lorensen_1987_marching_cubes,hoppe_1992_surface_reconstruction,kazhdan_2006_poisson}。这些方法擅长拟合平滑曲面的连续生成，但都不直接输出 B-rep——要把结果落到 STEP 文件，仍需经过等值面提取、曲面重建或重新拟合等后处理步骤，每一步都可能引入额外的几何与拓扑误差：等值面提取容易在尖锐特征处产生锯齿，表面重建必须自行猜测面边界与裁剪曲线，后续拟合若失败则只能保留为网格。因此，这类方法即便在视觉指标上表现优秀，距离“可被几何内核继续使用的实体”仍有一段工程距离，与本文以 BRepCheck 为终点的目标不直接重合。

\subsection{多边形网格自回归方法}

PolyGen\cite{nash_2020_polygen} 是较早将三维几何视为离散序列对象进行自回归生成的工作，它把三角网格拆成“顶点序列 + 面索引序列”两阶段，先用一个 Transformer 生成排序后的顶点坐标，再用另一个 Transformer 在已生成顶点上预测三角面的顶点索引。该设计首次说明，结构化几何完全可以像文本一样被 token 化，并以 next-token 预测的方式学习其长程依赖。它对本文的启发主要在于“几何要素直接序列化 + 自回归预测”这条思路，而它的局限恰好定义了本文的扩展方向：PolyGen 的表征仍停留在多边形网格层面，没有面参数化曲面、没有面—边的层级拓扑、也不区分共享边与孤立边，因此并不能直接产出工业可读的 B-rep。本文把这一思路向上推广一层，把生成对象从“顶点 + 面索引”扩展为“NURBS 面采样点 + 面索引 + 边采样点 + 邻接面索引”。

\subsection{CAD 操作序列生成方法}

DeepCAD\cite{wu_2021_deepcad} 没有直接对 B-rep 几何建模，而是把 CAD 设计的“过程”作为生成目标——将一个零件还原为草图—拉伸—布尔等参数化 CAD 操作序列，用基于 Transformer 的自回归模型在“CAD 操作字典”上输出可执行的命令序列，后续 SkexGen 进一步引入解耦码本，以增强 sketch-extrude 序列的表示能力\cite{pmlr-v162-xu22k}。这类方法的优势是生成结果天然可解释、可编辑：每一步都对应一个 CAD 内核命令，得到的 B-rep 由内核执行而非神经网络直接产出，几何细节因此具有更高的语义一致性。代价则在于路线本身的表达能力受限于操作字典的覆盖范围，对自由曲面、复杂裁剪和高度非平凡的拓扑结构难以直接表示——一旦目标零件用不到字典中已有的操作，模型就需要靠组合堆叠近似，仍然无法直接生成几何。

\subsection{草图拉伸解耦生成方法}

SkexGen 在 DeepCAD 风格的草图—拉伸表征基础上做了更细的分解：把草图分支与拉伸分支的表征分别投影到多组相互解耦的离散码本，再对每组码本独立训练自回归生成器，最后通过联合采样组装出完整的 sketch-extrude 设计序列。这一解耦让“画什么草图”和“如何把草图拉伸成三维特征”两个生成子问题在统计上相互松绑，提升了输出序列的多样性和组合性，但是表征终点仍然是离散码本，单个 token 对应一组高维潜变量的中心，几何精度的上界由码本规模决定，码本越小则细节越粗糙，码本越大则训练越困难。从生成对象的角度看，SkexGen 仍然属于“生成 CAD 操作脚本”而不是“直接生成 B-rep”，与本文路线相比仍然隔着一层 CAD 内核执行步骤。

\subsection{结构化 B-rep 生成方法}

与前述操作序列路线不同，BrepGen\cite{xu_2024_brepgen} 与 SolidGen\cite{jayaraman_2022_solidgen} 直接以 B-rep 本身作为生成目标，但具体策略不同。BrepGen 把 B-rep 的面、边、顶点先编码为带有拓扑约束的结构化潜变量集合，再借鉴扩散模型的逐步去噪思想\cite{ho_2020_ddpm}，在潜空间中联合采样几何与拓扑，最后由解码器还原为 B-rep。它在 DeepCAD 数据集上展示了较强的几何质量，但潜空间的编码—解码同样需要预训练几何编码器，几何精度依赖潜变量分布是否被准确捕获，潜空间过度光滑则细节模糊，过度多模态则采样不稳定。SolidGen 走的是更接近本文的自回归路线，直接对 B-rep 的顶点—边—面分层结构做序列生成，它论证了“无需操作字典也能在 B-rep 上做自回归”这一可能性，但生成对象仍以顶点—边的离散结构为主，对 NURBS 曲面采样点的连续几何表达较为有限。

\subsection{现有方法比较与研究定位}

将上述方法按“中间表征”维度作横向比较，可以看到现有路线大致可分为四类：CAD 操作序列（DeepCAD）、离散码本上的 sketch-extrude 解耦（SkexGen）、结构化潜空间 + 扩散（BrepGen）、以及无码本的 B-rep 自回归（SolidGen）。它们各自的中间表征——操作字典、码本索引、潜变量、顶点—边索引——都不是 B-rep 几何采样点本身，几何细节都需要再经过一次解码或内核执行才能落到坐标上。本文工作与上述方法的关键差异有两点。其一是不依赖显式 VQ-VAE 码本\cite{van_den_oord_2017_vqvae}或潜空间扩散，几何精度的下界由 4000 级标量量化粒度直接决定，便于解释和定位误差。其二是与 SolidGen 相比，本文显式地在序列中携带 NURBS 曲面 / 曲线的采样点而不仅是顶点坐标，使重建阶段能够直接拟合参数化曲面而无需额外的曲面恢复算法。这两点共同构成了“点级 NURBS 序列化 + 标准 next-token 自回归”的本文切入点。


\section{本文研究内容与主要工作}

本文围绕“不依赖显式码本、端到端生成可校验 CAD 实体”这一总体目标，按照“数据表示、序列生成、几何重建”三条线索展开研究。第一章只从整体思路上概括本文工作，具体的采样规则、模型结构和实验参数将在后续章节分别说明。

\begin{enumerate}
    \item 在数据表示方面，本文将 B-rep 实体转化为可被神经网络处理的序列。基本思路是保留面、边及其连接关系中的关键信息，把连续几何与离散拓扑放入同一条生成序列中，使模型学习的不只是零件外形，也包括各几何要素之间的组织方式。为减少同一实体在不同导出顺序下产生的差异，本文还引入确定性的排序思路，使训练样本具有更稳定的序列结构。
    \item 在生成模型方面，本文采用自回归 Transformer 作为主要建模框架。模型按照“已生成内容约束后续内容”的方式逐步生成 B-rep 序列，避免把几何信息先压缩到不可解释的离散码本中。这样设计的出发点不是追求复杂网络结构，而是先验证一种直接面向 B-rep 的序列生成路线是否能够跑通，并观察它在 CAD 数据上的工程可行性。
    \item 在重建与评测方面，本文把生成结果重新送回几何内核进行检验。生成序列只有能够被解析、拟合、缝合，并最终写出可校验的 STEP 文件，才说明该生成结果具备进一步编辑和工程使用的基础。因此，本文不只关注模型损失是否下降，也关注生成结果能否通过 OpenCASCADE 的合法性检查。
\end{enumerate}

上述三方面构成了一条从 CAD 数据到生成结果再到内核校验的完整链路。数据表示解决“模型看什么”的问题，生成模型解决“模型如何产生新序列”的问题，逆向重建与评测解决“生成结果是否真的可用”的问题。本文后续章节将沿着这一逻辑展开，第~3 章重点说明数据如何组织，第~4 章说明模型和重建流程如何实现，第~5 章再用实验结果检验整条路线的有效性。


\section{论文组织结构}

全文按以下五章组织：第~1 章交代研究背景、问题定义、国内外现状与主要工作，第~2 章介绍 B-rep / NURBS 几何、自回归序列建模、LLaMA3 关键组件和 CAD 生成评价指标，第~3 章给出 B-rep 序列化与数据预处理方案，包括面 / 边采样、坐标量化、统一词表与拓扑感知排序，第~4 章描述自回归生成模型架构、训练 / 推理流程以及序列到 STEP/STL 的逆向重建管线，第~5 章在 DeepCAD 子集上汇报训练收敛性、生成有效率与三态分布，讨论局限并给出结论与展望。
